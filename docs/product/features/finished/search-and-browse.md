# Search And Browse

`mdmind` now treats finding things as one surface with three modes:

- `Query`
- `Browse`
- `Saved Views`

Open them with:

- `/` for query
- `b` for browse
- `w` for saved views

How they differ:

- `Query` is for “I know what I want”
- `Browse` is for “show me what exists”
- `Saved Views` is for “reopen a working set”

Browse is especially useful because it exposes:

- tags
- metadata keys
- metadata values
- ids / deep-link targets

That means users do not need to remember the exact vocabulary of a map before they can navigate it well.

Good uses:

- find all `#todo` or `@status:blocked` work
- inspect which `owner` values exist in a map
- jump straight to a deep-linked branch from the `Ids` tab
- save a recurring review filter and reopen it later
