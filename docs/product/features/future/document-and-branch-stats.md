# Document And Branch Stats

This is a small feature with good leverage if it stays restrained.

Useful stats:

- row count
- detail-word count
- detail-character count
- maybe counts for tagged or filtered items

Why it could matter:

- writing and research maps benefit from a sense of volume
- review workflows benefit from counts
- branch-level stats could make focused work feel more concrete

Where it should live:

- status/help/inspection surfaces
- maybe a palette or info overlay

Where it should not live:

- always-on chrome everywhere
- raw file syntax
- a dashboard-heavy secondary product

Complexity guardrail:

Stats should help orientation, not turn the app into analytics. If they need a lot of explanation, they are too heavy.
