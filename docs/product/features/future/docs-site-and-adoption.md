# Docs Site And Adoption

The repo docs are much healthier now, but a public docs site is still future work.

What already exists:

- shorter `README.md` front door
- curated docs portal
- feature shelves
- built-in help
- examples gallery
- user-facing guides for outliner, agent, query, ids, relations, details, templates, and workflows

What is still future work:

- a real public docs site
- screenshot-heavy walkthroughs and demos
- more polished “why mdmind” product framing outside the repo
- broader adoption material for users who will never read the full docs tree

This should stay on the future shelf until there is an actual docs-site implementation plan, not just better repo docs.
