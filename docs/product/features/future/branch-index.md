# Branch Index

This is the “section list for large maps” feature.

It should not be a view mode.

Why:

- view modes answer “what is visible?”
- a branch index answers “where do I want to jump?”

That means this should be a navigation surface, not a projection mode.

Good forms:

- a palette shelf
- a quick-jump overlay
- maybe a lightweight side surface later

What it should do:

- list major branches and sections
- make large maps easier to scan and jump through
- pair well with recent places and saved views

What it should not do:

- become another permanent pane by default
- compete with search and palette
- rewrite the meaning of view modes

Complexity guardrail:

This feature only works if users can still say, in one sentence, what it is for: “jump around a large map by section.”
