# Outliner Guides And Help

`mdmind` is a mind map tool, but it is also a very usable outliner.

That should be taught more directly.

What this work means:

- a built-in help section about using `mdmind` as an outliner tool
- clearer docs for people who think in outlines more than in maps
- better onboarding copy for writing, planning, research, and checklist-style use cases

Why this belongs in `inwork`:

- the docs and built-in help system already exist
- this is more about shaping the product story and guidance than inventing a new technical system

Good outcomes:

- users understand that `mdmind` is tree-first and outline-friendly
- fewer people assume they need the visual mindmap mode before the product is useful
- detail lines, reading mode, focused views, and minimal mode make more sense in the outliner framing
