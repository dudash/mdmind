# Vision

`mdmind` is a local-first, terminal-native tool for turning messy thought into structured maps.

The world has plenty of tools for documents, kanban boards, and visual canvases. What is missing is a tool that lets a technical thinker stay in flow while moving between:

- raw ideas
- requirements
- TODOs
- prompts
- linked concepts
- implementation plans

without breaking stride.

## Problem

Most tools force a false choice:

- plain text is fast but structurally weak
- diagram tools are expressive but slow to edit
- docs are linear while ideas are not
- task tools track work but not thought

Developers, product builders, and AI-assisted makers often start with fragments:
a feature idea, a prompt tweak, a constraint, a follow-up question, an implementation note.

Those fragments want to become a tree before they become a spec.

## Vision

`mdm` should feel like sketching a project directly in your terminal.

`mdmind` should feel like diving into that project and shaping it with precision.

Together they should make it easier to:
- explore a product idea
- define requirements
- manage TODOs
- refine prompts
- keep context connected over time

## Who it is for

- developers
- product managers
- founders
- design-minded builders
- AI-native makers working in IDEs and terminals

## Product thesis

The right abstraction is not “document” or “diagram”.

It is a living, structured tree of thought that stays readable as plain text and becomes powerful through:
- filtering
- deep links
- metadata
- exports
- focused views

## Non-goals

- Not a collaborative SaaS product 
- Not a full graphical whiteboard
- Not a database-backed note system
- Not a plugin platform on day one
- Not an AI wrapper with shallow gimmicks

## Success signal

Someone working on a feature, product idea, or prompt library reaches for `mdm` first because it is the fastest way to get structure out of their head.
