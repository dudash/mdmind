# Using mdmind As An Outliner

`mdmind` is a mind map tool, but it is also a very good outliner.

If you are coming from OmniOutliner, Workflowy, Logseq outlines, or just plain nested notes, this is the best way to think about it:

- one line is one row
- children create hierarchy
- the tree is the main working surface
- search, palette, and focused views help you stay calm as the outline grows

You do not need the visual mindmap on day one.

## The Simple Mental Model

Start here:

- move through rows with `‚Üë / ‚Üì`
- expand or collapse structure with `‚Üê / ‚Üí` and `Enter`
- add and edit rows with `a` and `e`
- use `d` when one row needs longer notes
- use `/` when the outline gets noisy
- use `:` when you already know what you want

That is already a solid outlining workflow.

## Why mdmind Works Well As An Outliner

### 1. The Tree Stays Primary

The product still feels like an outline first.

- view modes change projection, not the document
- minimal mode reduces chrome
- reading mode emphasizes long-form detail content
- details keep long notes attached without bloating row labels
- the palette helps you jump without opening ten side panels

### 2. It Scales Better Than A Plain Nested Note

As the outline gets larger, you do not have to rely on memory.

You can use:

- search for text, tags, or metadata
- browse to inspect tags, values, and ids
- saved views for recurring review slices
- focused views to work inside one branch without manual collapse chores

### 3. It Holds More Than Short Labels

Node details are the key feature here.

Use them for:

- meeting notes
- writing notes
- rationale
- quotes
- research excerpts

That lets the visible outline stay clean while still holding real content.

## Best Outliner Setup

Once the basics feel natural, a very calm setup is:

1. use `Focus Branch` or `Subtree Only`
2. turn on `minimal mode`
3. turn on `reading mode`
4. use node details for longer text

That combination keeps the interface light while turning the selected node into a calmer inline document-like reading surface inside one branch.

## What To Ignore At First

If you are primarily using `mdmind` as an outliner, you can safely ignore these at first:

- ids
- relations
- checkpoints
- visual mindmap

They are useful later, but they are not required for normal outlining.

## When The Mindmap Helps

The visual map is still useful sometimes.

Good times to use it:

- presenting a branch
- checking cluster shape
- reviewing cross-links

But it should feel like a second lens, not the main tool.
