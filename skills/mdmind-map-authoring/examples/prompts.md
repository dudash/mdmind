# Example Prompts

Use these prompts to test triggering and output quality for `mdmind-map-authoring`.

## Prompt 1: Meeting Notes To Map

Turn these notes into a native mdmind map.
Keep the tree easy to scan.
Use `#todo` for follow-up work, `#decision` for decisions, and `@owner` or `@status` only when the notes support them.
Add ids only to durable branches.
Use details for rationale or meeting context that does not belong in labels.

## Prompt 2: Research Synthesis

Convert this research dump into an mdmind map with themes, evidence, open questions, and next actions.
Keep labels short.
Use details for quotes and nuanced findings.
Use ids on major themes only.

## Prompt 3: Existing Map Cleanup

Restructure this mdmind file so it reads more like a clean outline.
Shorten verbose node labels, move nuance into detail lines, preserve useful tags and metadata, and remove unnecessary ids or relations.
