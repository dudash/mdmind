Local-first. Plain text. Keyboard-first.

Structured thinking that still lives in normal Markdown files.

mdmind gives you two interfaces over the same map: mdm for CLI inspection and export, and mdmind for a calm full-screen TUI. It is built for outlines, plans, writing maps, research trees, prompt libraries, and large idea structures that need to stay searchable and safe to edit.

Why it feels different

Not a whiteboard clone. Not a note dump.

mdmind keeps the tree primary. It uses lightweight syntax for structure, view modes to reduce noise, and a real safety layer so large maps feel editable instead of fragile.

Normal Markdown shape

Core features

The things that matter in daily use

Map language

Tags, metadata, ids, and cross-links inside plain text

Use #tags for grouping, @key:value for structure, [id:...] for stable deep links, [[target]] for references, and | details for longer notes attached to a branch.

Navigation

View modes and section jumps

Work in Full Map, Focus Branch, Subtree Only, or Filtered Focus, then jump by major branch through the command palette’s section index.

Search

Query, browse, saved views

Search by text, browse existing tags, metadata, and ids, reopen saved working sets, and use recipes when you know the workflow but do not want to remember the filter.

Relations

Backlinks and lateral structure

Add cross-branch references without turning your map into a graph-first mess. Follow outgoing links, jump by backlink, and see relation edges in the visual map.

Details

Long-form notes without bloating labels

Keep node titles short. Attach rationale, quotes, scene notes, meeting context, or research snippets underneath the branch they belong to.

Visuals

Visual mindmap as a second lens

Open the current working set as a visual mindmap, keep it aligned with your active view and filter, and export a polished PNG when the tree needs a presentation surface.

Safety

Undo, redo, checkpoints, restore history

Large structural edits stay reversible. mdmind includes a real recovery layer, not just “hope you saved before that.”

Surface

Themes, minimal mode, reading mode

Switch from a fuller guided surface to a quieter expert layout, tune the theme, and expand long-form notes inline when you want a more editorial outliner feel.

Agent-friendly

Useful for AI-generated handoff maps, not just human typing

mdmind works well when agents need to produce structured outlines for humans to refine. The format gives agents a small, explicit vocabulary while staying readable and durable for the person who opens the file later.

Focused views

Large maps stay navigable without becoming a mess.

Full Map, Focus Branch, Subtree Only, and Filtered Focus give you different working scopes over the same file. The command palette can also jump by major branch through the section index when the map gets large.

Reading mode

Long-form notes stay attached to the branch they belong to.

Reading mode expands detail content inline instead of turning the outline into a separate editor. That makes writing, research, scene notes, and rationale feel natural without losing structure.

Second lens

Visual maps are there when the tree stops being enough.

The visual mindmap follows your current scope instead of inventing a separate truth. Use it for cluster recognition, presentations, and understanding branch shape.

CLI + TUI

One map file. Two strong surfaces.

  • mdm validates, views, filters, exports, and copies examples.
  • mdmind handles navigation, editing, search, views, help, and visual maps.
  • Both operate on the same plain-text file format.
Open and free
